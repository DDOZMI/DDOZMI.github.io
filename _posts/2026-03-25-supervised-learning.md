# 지도 학습

## 개요
>과제 T는 입력 $x∈X$에서 출력 $y∈Y$로의 매핑을 학습하는 것
출처 입력

이게 무슨 의미일까 생각해보자.

단순하게 생각하면 입력에서 출력 예측을 연결지어 학습한다는 것일까?
여기서 $x$와 집합$X$, $y$와 집합 $Y$는 무슨 의미일까?

$X$는 Input Space로, **모델이 받아들일 수 있는 모든 가능한 입력 데이터의 집합**을 의미한다.

단순히 "입력할 데이터셋"이라고만 생각할 수 없다. 수학적으로 $X$는 모델이 이론적으로 받을 수 있는 세상의 모든 데이터 범위라고 생각하자.

$Y$는 Output Space로, **모델이 출력할 수 있는 모든 가능한 결과의 집합**을 의미한다. $y$는 당연히 특정 입력 $x$에 대응하는 정답 1개를 의미한다.

예시를 들어 보자.
1. 스팸 메일 분류 (분류 문제)
$X$ : 세상에 존재할 수 있는 모든 이메일 내용의 집합
$x$ : 방금 내 메일함에 도착한 특정 이메일 한 통의 내용
$Y$ : {스팸, 정상} (출력할 수 있는 모든 경우의 수)
$y$ : 스팸 (내가 받은 특정 메일의 정답)

2. 아파트 가격 예측 (회귀 문제)
$X$ : 세상 모든 아파트의 특징들(면적, 층수, 위치 등)의 조합
$x$ : '서울시 강남구 30평, 중층, 방 3개'라는 특정 아파트 하나의 정보
$Y$ : 예측 가능한 가격의 범위(모든 실수 값의 집합)
$y$ : 30억원 (실제 이 아파트의 가격)

### 요약
지도 학습에서 Mapping $f : X → Y$를 학습한다는 것은, 무수히 많은 입력 데이터 $x$와 그에 해당하는 정답 $y$들의 쌍(Training dataset)을 모델에게 보여주면서, 새로운 $x$가 들어왔을 때 가장 알맞은 $y$를 찾아내는 규칙 $f$를 찾으라고 시키는 과정이다.

## 분류 / Classification

출력 공간은 Class라 하는, 순서 없는 상호 배반적인 label $C$의 집합 $Y = {1, 2, ... , C}$ 이다.

Class는 분류가 가능한 카테고리인 것은 직관적으로 감이 온다. 하지만 그 뒤의 내용은 무슨 소리인지 도통 알 수가 없다. 하나 하나 뜯어봐야겠다.

1. **상호 배반적인 / Mutually Exclusive**

정답이 무조건 딱 하나만 있어야 한다는 뜻이다.

예로 사진 속 동물이 '개'인지 '고양이'인지 맞히는 문제를 생각해 보자.

어떤 사진이 주어졌을 때, 이 사진은 '개'이면서 동시에 '고양이'일수는 없다. 반드시 둘 중 하나로 구분되어 속해야 한다. 

이처럼 정답 카테고리들이 서로 겹치는 영역 없이 명확하게 나뉘어 있을 때 통계학에서 이를 '상호 배반적'이라고 부른다.

2. **순서 없는 / Unordered**

카테고리 사이에 크기, 우열, 순위가 존재하지 않는다는 뜻이다.
예로 개, 고양이, 새를 분류한다고 해 보자.

이들 중 어느 하나가 더 큰 값을 가지거나, '새'가 1등 이라는 식의 순서가 성립하지 않는다. 이들은 그저 서로 다를 뿐인 것이다.

반면, '순서가 있는' 경우도 있다. 예를 들어 영화 평점(1점~5점)이나 옷 사이즈(S, M, L)은 명확한 순서와 크기 차이가 있다.

Classification 문제에서 다루는 Class는 보통 이런 순서가 없는 명목형 데이터를 말한다.

3. **그래서 $Y = { 1, 2, ... , C }$ 는 뭔데?**

컴퓨터는 '개', '고양이'같은 글자보다 숫자를 더 잘 이해한다. 그래서 이 Class에 번호표를 부여한 것일 뿐이다.

만일 동물 3종류(개, 고양이, 새)를 분류한다면 $C = 3$이 된다.
컴퓨터에게 입력하기 위해 '개'는 1, '고양이'는 2, '새'는 3이라고 Label을 붙인 것이다.

이렇게 되면 모델이 출력할 수 있는 정답의 집합은 $Y = {1, 2, 3 }$ 이 되는 것이다.

※ 여기서 $1, 2, ... , C$는 수학적인 크기가 아니다. 그저 등번호와 같은 이름표의 역할이다!

### 요약
Classification 문제에서 모델이 뱉어내는 정답 $Y$는, 개나 고양이처럼 서로 겹치지 않고(상호 배반적), 크기나 순위 비교가 불가능한(순서 없는) 카테고리들에 단순히 $1, 2, ...$ 의 번호표를 붙여 모아둔 집합이다.

책에서는 R데이터분석을 들으면서 자주 봤던 붓꽃 데이터셋이 나왔다.

이미지 분류에서 입력 공간 X는 이미지의 집합이고 차원이 매우 높은 공간이다.

예시를 들면 채널 $C = 3$이고 (RGB, 실제로는 각 채널마다 0, ..., 255이지만 생략) $D_1 \times D_2$ 픽셀인 컬러 이미지에서 $X = ℝ^D$ 이고, $D = C \times D_1 \times D_2$ 이다. 이런 이미지에서 라벨로의 매핑 $f : X → Y$를 학습하는 것은 어려운 일이다.
이 데이터셋에서는 꽃밭침 길이, 꽃밭침 너비, 꽃잎 길이, 꽃잎 너비를 이미 식별해 놓았다.
이 경우 $X = ℝ^4$ 라고 볼 수 있다. 이렇게 작은 수의 특성 데이터셋이 있을 때는 $N \times D$ 행렬에 저장하는게 일반적이다.

**$N \times D$ 행렬은 또 뭐지?**

$D$가 Dimension이라는 것은 알겠다. 그럼 여기서는 4가 될 것이다.

**$N$과 $D$의 의미**

- $D$(차원, 특성 수) : 데이터가 가진 특징(Feature)의 개수이다. Iris 데이터셋에서는 특징이 4개이므로 예상과 동일하게 $D = 4$이다.
- $N$ (Number of Samples) : 예제 수라고 하니 이해가 안 가지만 샘플의 개수, 즉 수집한 전체 데이터의 총 개수이다. Iris 데이터셋은 총 150송이의 붓꽃을 측정한 데이터이므로 $N = 150$이 된다.
즉, $N \times D$ 행렬은 150행과 4열로 이루어진 엑셀 표라고 생각하면 된다.

| | Sepal Length | Sepal Width | Petal Length | Petal Width | label |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **Iris 1번** | 5.1 | 3.5 | 1.4 | 0.2 | setosa |
| **Iris n번** | ... | ... | ... | ... | versicolor |
| **Iris 150번** | 5.9 | 3.0 | 5.1 | 1.8 | virginica |

위 표처럼 행은 붓꽃 한 송이의 완전한 정보를 담고 있다. 열은 각각의 값들을 가지고 있다.

이걸 Design Matrix(디자인 행렬)이라고 한다.
또 들어본 적 없는 단어다. 앞선 사례처럼 모든 입력 데이터 $X$를 $N \times D$ 의 표 형태로 모아놓은 전체 덩어리를 부르는 정식 명칭이 Design Matrix라고 한다. 그리고 앞선 붓꽃 데이터셋과 같은 형태를 Tabular Data(표 데이터)라고 한다.

$N >>> D$인 것처럼 모양이 길고 좁으면 Big Data, $D >>> N$처럼 짧고 두꺼우면 Wide Data라 한다고 설명하고 있다.

입력이 고정된 길이가 아닌 가변 크기 벡터면 디자인 행렬이 아니라 다른 형식으로 저장하고, 그런 데이터는 고정된 크기의 특성 표현으로 변환되는 경우가 많으며(특성화), 추가적인 처리를 위해 디자인 행렬을 암묵적으로 만든다.

나오는 문장마다 이해가 가지 않는다. 이것도 하나 하나 전부 뜯어보자.

붓꽃 데이터는 특성이 4개로 고정되어 있다. 하지만 실생활에서 쓰는 데이터는 그렇지 않은 경우가 많다.

**예: 텍스트 데이터**
- data 1 (짧은 문장) : 나는 / 고양이가 / 좋다 (3 단어)
- data 2 (긴 문장) : 나는 / 어제 / 길을 / 걷다 / 귀여운 / 고양이를 / 보아서 / 기분이 / 좋았다 (9 단어)

**여기서 왜 디자인 행렬을 못 쓸까?**

앞서 디자인 행렬은 $N \times D$ 형태의 직사각형 표라고 이해했다.
어떤 행은 3칸만 채워져 있고, 어떤 행은 9칸이 채워져 있다면 이것으로는 직사각형 표를 만들 수가 없다.

**그럼 어떻게 저장해?**

억지로 표에 넣지 않고 텍스트 문서 또는 리스트 형태로 원래의 들쭉날쭉한 형태를 유지하고 저장해 둔다.

**고정된 크기의 특성 변환으로 변환 (특성화 / Featurization)**

문제는 머신러닝 모델은 무조건 크기가 고정된 숫자(직사각형 표)만 받아들일 수 있다는 점이다.

이런 들쭉날쭉한 데이터를 모델이 소화할 수 있는 고정된 규격으로 가공해주는 작업이 필요한데, 이를 특성화(Feturization) 또는 특징 추출이라고 부르는 것이다.

**어떻게 규격화하지?**

세상에 존재하는 주요 단어 10,000개를 모아 단어장을 만든다. 그리고 각 문장에 단어장의 단어가 몇 번 들어갔는지 세어보는 것이다.

- data 1 : [ 나 1 , 고양이 1 , 좋다 1 , 강아지 0 , 자동차 0 , ... (총 10,000칸) ]
- data 2 : [ 나 1 , 고양이 1 , 좋다 1 , 걷다 1 , 귀엽다 1, ... (총 10,000칸) ]
- 결과는?
원래 길이가 3, 9로 달랐던 두 데이터가 이제는 둘 다 똑같이 10,000칸짜리 고정된 길이의 숫자 데이터로 변한 것이다. (D = 10000이 된 것)

**암묵적으로 디자인 행렬을 만든다? (Implicit Design Matrix)**

이제 데이터 길이가 같아졌으니 다시 1번부터 끝까지 차곡차곡 쌓아 N x 10000 짜리 거대한 디자인 행렬을 만들면 된다.

**여기서 암묵적이라는건?**

단어 10000칸 중 대부분은 한번도 쓰이지 않은 단어라 0으로 채워져 있을 것이다. 이걸 모든 문장에 대해 엑셀 표처럼 RAM에 다 만들어버리면 불필요한 0때문에 컴퓨터가 감당할 수 없는 크기가 될 수 있다.

**해결책**

저 메모리를 통짜로 올리기보다는 "1번 문장은 3번째, 5번째, 10번째 칸만 1이고 나머지는 전부 0이다" 식으로 요약 저장(희소행렬 / Sparse Matrix) 하거나, 모델이 학습할 때마다 필요한 만큼만 조금씩 잘라서 표를 만든다.

즉, 수학적으로는 거대한 디자인 행렬이 있다고 가정(암묵적)하고 계산을 하지만, 물리적으로 컴퓨터 메모리 상에 이러한 표를 통째로 만들지는 않는다는 뜻이다.

## 탐색적 데이터 분석 / Exploratory Data Analysis

1. **ML로 문제를 보기 전에 EDA를 수행하라**

컴퓨터에게 무작정 데이터를 던져주기 전에, 먼저 눈으로 데이터를 훑어보라는 뜻이다. 이 과정을 탐색적 데이터 분석(EDA, Exploratory Data Analysis)라고 부른다.

여기서는 크게 두 가지를 본다.

- 명백한 패턴 찾기 (모델링 힌트) : 데이터를 그렸더니 붓꽃 종류별로 선을 하나 쭉 그어 나눌 수 있을 것 같다면 굳이 무겁고 복잡한 딥러닝 모델을 쓸 필요 없이 단순한 선형 모델을 선택하면 된다. 즉, 어떤 방법론을 꺼낼지 힌트를 얻는 과정이다.

- 명백한 문제 찾기 (이상치 / Outlier) : 사람 나이 데이터인데 '999살'이 들어있거나 꽃잎 길이가 '100m'로 적혀있는 등 상식적으로 말이 안되는 튀는 값을 찾는다.

- 라벨의 잡음(Label Noise) : 분명히 고양이 사진인데 정답이 '강아지'로 잘못 달려 있는 경우를 말한다. 이런 불량 데이터는 미리 걸러내야 한다.

2. **산포도(Scatter Plot)와 쌍 도표(Pair Plot)**

데이터를 어떻게 눈으로 볼까? 앞서 언급된 $N \times D$ 행렬에서 특성 수(D)가 적은 표 데이터라면 그래프를 그려보는 것이 최고다.

- 산포도(Scatter Plot) : 가로축에 '꽃잎 길이', 세로축에 '꽃잎 너비'를 두고 데이터 1개당 점 하나씩을 찍어 그린 그래프다. 점들이 어떻게 흩어져 있는지 볼 수 있다.

- 쌍 도표(Pair Plot) : 특성이 4개라면, 가능한 모든 두 특성의 짝(Pair)을 지어서 여러 개의 산포도를 하나의 바둑판 같은 표로 모아놓은 것. (모든 순서쌍에 대해 도표를 그려보자)

3. **대각 요소는 주변 밀도를 보여준다는건 뭘까?**

바둑판 모양의 쌍 도표를 생각해보자.
행도 특성, 열도 특성이다.

- 대각선이 아닌 칸 $(i, j)$ : 가로가 1번 특성, 세로가 2번 특성이라면 두 특성의 관계를 점으로 찍은 산포도가 들어간다.
- 대각선 칸 $(i, i)$ : 대각선 위치는 가로가 1번 특성, 세로도 1번 특성이다. 같은 특성끼리 산포도를 그리면 그냥 45도의 무의미안 직선이 되어 버린다.
- 주변 밀도(Marginal Density) : 같은 특성이 만나는 칸에는 산포도 대신 "그 특성 하나만 놓고 봤을 때 데이터가 어디에 많이 몰려있는지"를 보여주는 히스토그램 그린다. 이걸 통계 용어로 '주변 밀도'라고 부른다. 다시 말하면 각각의 특성(sl, sw, pl, pw) 하나만 가지고 각 Class(Setosa, Versicolor, Verginica)가 어떻게 분포하는지 보는 것이다. 각 붓꽃 종류들이 하나의 분류 기준으로 보았을 때에는 어떻게 위치할까에 대한 것.

4. **고차원 데이터의 "차원 축소 후 2차원/3차원 시각화"는 뭘까?**

인간이 가진 한계와 데이터의 크기 때문에 등장하는 개념이다.

- 고차원 데이터란? : 붓꽃 데이터는 특성이 4개라서 $4 \times 4$ 도표는 사람이 보기 편하다. 그러나 특성이 1000개인 데이터라면? 1,00,000칸짜리 표를 볼 수는 없다.
- 시각화의 한계 : 4차원 이상의 그래프는 볼 수 있게 그릴 방법이 없다.
그래서 차원 축소(Dinensionality Reduction)을 사용한다.
그림자라고 생각하자. 3차원 입체 모형에 스크린을 비춰 2차원 평면 그림자로 만들면 2차원으로 줄어들었지만 해당 물체의 원래 정체를 파악할 수가 있다. 차원은 줄어들었어도 그 특징을 가지고 알아챌 수 있는 것이다.

### 요약
AI 쓰기 전에 데이터에 이상치가 있는지 없는지, 어떤 모델을 쓰면 좋을 지 미리 파악하기 위해 특성들끼리 짝을 지어 점을 찍어보는 그래프(쌍 도표)를 그려 눈으로 확인해라.

## 분류기 학습
$$f\left(x\ ;\ \theta \right)\ =\ \begin{cases}꽃잎\ 길이\ <\ 2.45인\ 경우,\ Setosa\\그\ 이외의\ 경우,\ Versicolor\ 또는\ Verginica\end{cases}$$
위와 같은 결정 규칙을 만든다고 해 보자.

입력 공간 $x_{꽃잎길이} = 2.45$ 에서 1차원 결정 경계(Decision Boundary)로 정의한 2개 영역으로 분할한다.

이 경우에 Setosa에 대해서는 완벽하게 분류하지만 Versicolor나 Verginica에 대해서는 그렇지 못하다.

성능 개선을 위해 추가로 분할을 해볼 수도 있다.

위 테스트에서 실패한 입력에 적용하는 다른 의사결정 규칙을 추가하여 둘을 분류할 수 있다. 이렇게 중첩된 규칙을 의사결정 트리(Decision Tree)라고 부르는 트리 구조로 배열할 수 있다.

## 경험적 위험 최소화

지도 학습의 목표는 분류 모델을 자동적으로 만들어 어떠한 주어진 입력에 대해 유의한 예측을 할 수 있도록 하는 것이다. 이러한 과제의 성능을 측정하는 일반적인 방법은 훈련 집합에서의 오분류율(Misclassificaion rate) 측면으로 하는 것이다.

1. 오분류율 (Misclassification Rate)

의미 : 쉽게 말해서 "오답률". 모델이 100문제 중에서 몇 개를 틀렸는지 그 비율을 보겠다는 뜻이다.

2. 수식 해독
$$L(\theta) \triangleq \frac{1}{N} \sum_{n=1}^{N} \mathbb{I}(y_n \neq f(x_n; \theta))$$

이게 뭔 소린지 하나씩 번역해보자.
- $L(θ)$ : Loss, 즉 총 벌점이다. θ 는 모델의 뇌 상태를 의미하고 저 세모등호는 "오른쪽처럼 정의한다"는 의미다.
- $II$ ( 조건) : 지시 함수(Indicator function)이라고 부르는데, 괄호 안의 조건의 참이면 벌점 1점, 거짓이면 0점을 준다.
- (조건) : $y_n$ 은 '실제 정답' 이고, 나머지 $f$는 'AI가 찍은 답'이다. 즉, 정답과 AI의 예측이 다르다(틀렸다)라는 뜻이다.
- $\frac{1}{N}\sum$ : 1번 문제부터 $N$번 문제까지 다 더한 다음, 전체 문제 수인 N으로 나눈다는 뜻이다. 즉, 평균을 내는 과정이다.
한줄요약: 1번부터 $N$번까지 채점(II) 해서 틀릴 때마다 벌점 1점씩 주고, 그걸 다 더해서 평균을 낸 것이 총 벌점($L$)이다. = 단순 오답률

3. 비용(Cost)와 비대칭 손실 함수의 의미
여기서 비용(Cost)는 틀렸을 때 치러야 할 대가(치명도)를 말한다.
앞서 말한 지시 함수$II$는 모든 오차를 동등하게 취급한다. 즉, 1번 문제든 100번 문제든 틀리면 무조건 벌점 1점.
이걸 Zero-one loss라고 부른다. 맞으면 0점, 틀리면 1점이라는 뜻이다.
하지만 현실에서는 모든 실수의 대가가 같지 않다!

책에서 든 예시로, 야생에서 붓꽃을 찾았는데 Setosa, Versicolor는 맛있지만 Virginica는 독이 있다고 해 보자.

| 실제 붓꽃 종류 | AI 예측 결과 | 오답 결과 (비용) | 채점 방식 |
| :--- | :--- | :--- | :--- |
| Setosa (맛있음) | Versicolor (맛있음) | 맛은 다르지만 안 죽음 | 벌점 1점 |
| Virginica (독초) | Setosa (맛있음) | 맛있는 줄 알고 먹었다가 죽음 (치명적) | 벌점 10,000점 |

이처럼 "어떤 종류로 틀렸느냐"에 따라 벌점(비용)을 다르게(비대칭적으로) 매기는 채점 방식을 비대칭 손실 함수(Asymmetric loss function)이라고 한다. 독초를 먹는 치명적인 실수를 하면 벌점을 왕창 매기는 식.

요약하자면, 분류 모델이 얼마나 똑똑한지 보려면 '오답률'을 보면 된다. 그런데 현실에서는 치명적인 오답이 있을 수 있는데, 그럴 때에는 단순 오답률만 보는 것이 아닌 치명적인 실수에 벌점을 왕창 때리는 비대칭 손실 함수를 써야 한다.

4. **경험적 위험 = 오분류율?**
- 경험적 위험(Empirical Risk) : 단순하게 "우리가 가진 훈련 데이터"를 가지고 채점해 본 평균 벌점이라는 뜻. "네가 만약 틀릴 때마다 무조건 1점씩 주는 zero-one loss를 썼다면, 그 평균 벌점(경험적 위험)은 결국 오답률(오분류율)이랑 똑같은 것 아니냐?" 라는 소리.

5. **경험적 위험 최소화 / Empirical Risk Minimization**
- $θ$ (모수, parameter) : 모델의 "뇌 구조 설정값". 정답을 맞히기 위해 조절할 수 있는 수많은 설정값.
- $arg min$ : 오른쪽에 있는 값(벌점)을 최소화하는 입력값을 찾으라는 명령문.
- $\hatθ$ : 찾은 최적의 정답을 의미.
수식을 요약하자면 설정값($θ$)를 변경해가면서 가지고 있는 훈련 데이터(경험)에서 발생하는 평균 벌점($L$)이 가장 0에 가까워지도록 세팅하자. 그리고 그 최적의 세팅값이 $\hatθ$이다.

즉, 훈련 데이터셋을 학습하면서 Loss가 최소화되는 방향으로 파라미터를 조정하는 것을 말한다.

6. **일반화 / Generalization**
앞선 경험적 위험 최소화가 중요하긴 하지만 진짜 목표는 학습 데이터 외의 다른 데이터에서의 예측 손실 최소화이다.

즉, 이미 많이 본 데이터에서 잘 맞히기보다는 전반적으로 일반화하기를 원한다.

기출문제 전부 외워 100점 맞는 것만이 아닌 새 문제를 보고도 잘 맞추기를 원하는 것.

## 불확실성

모델이 정답을 100% 확신하지 못하는 이유는 크게 두 가지이다. "내가 모르는 것"과 "원래 모르는 것".

1. **인식론적 불확실성(Epiostemic / 모델 불확실성)**
모델이 아직 공부가 부족해서 모르는 것.
예시: 처음 보는 붓꽃에 대해서는 본 적이 없어서 햇갈리는 상황.
해결책: 이 붓꽃 데이터를 더 많이 수집해서 학습시키면 불확실성은 줄어든다.

2. **우연적 불확실성(Aleatoric/데이터 불확실성)**
세상에 존재하는 기본적인 '운' 또는 '노이즈'.
예시: 주사위를 던졌을 때 무슨 숫자가 나올지 맞히는 상황.
해결책: 데이터를 아무리 모아도 절대 예측할 수가 없다. 원래부터 랜덤이니까.

## 조건부 확률

모델이 "이건 100% 안전한 붓꽃이야" 라고 단정짓는 것은 위험하다.

앞서 말한 불확실성 때문. 따라서 모델은 확률($p$)로 대답해야 한다.

위 식을 풀어 설명하면 "주어진 데이터 $x$와 내 뇌 구조 $θ$를 바탕으로 생각해 봤을 때, 이 꽃이 $c$번 종류일 확률은 $f$ 함수로 계산할 수 있다"는 뜻이다.

- 소프트맥스(Softmax)와 로짓(Logit) - 점수를 확률로 만들자!

모델이 계산한 결과값은 보통 제멋대로인 숫자가 튀어나온다.

이 날것의 숫자를 로짓(Logit, a)라고 부른다.

하지만 우리가 원하는 것은 확률이다. 확률이 되려면 두 가지 조건을 만족해야 한다.

1) 모든 값은 0과 1사이여야 한다. $(0 <= f_c <= 1)$

2) 모든 확률은 다 더하면 무조건 1(100%)가 되어야 한다. $( Σ f_c = 1 )$

이 제멋대로인 로짓(점수)들을 확률 조건에 우겨넣어주는 필터가 바로 소프트맥스(Softmax) 함수이다.

정답에 가까운 큰 숫자는 1에 가깝게 확 밀어주고, 나머지 찌꺼기들은 0으로 팍 눌러버리는 역할이라고 생각하자.

$$\text{softmax}\left(a\right)_c\ =\ \frac{e^{a_c}}{\sum _{c'=1}^Ce^{a_{c'}}}$$

*여기서 궁금증. $e$는 왜 쓸까? (나중에 다룰 것 같은데 그냥 궁금해서 먼저 정리해본다)*

1. 음수를 양수로

구하고 싶은건 확률인데 로짓 a는 음수일 수도 있다. 이걸로는 비율을 계산할 수 없으므로 e의 지수로 올려서 아무리 큰 음수라도 무조건 0보다 큰 양수로 변환시킨다. $( e^{-n} = \frac{1}{e_n} )$

2. 1등 밀어주기

e를 사용하는 지수 함수는 값이 조금만 커져도 결과값이 폭발적으로 커진다.


3. 미분이 편해서

나중에 경사하강법(Gradient Descent)을 한다고 하면 미분을 엄청 해야 하는데 $e^x$ 는 미분해도 그대로 $e^x$ 라 계산이 매우 편해진다.

덕분에 편미분을 반복하는 역전파(Backpropagation) 과정에서의 계산이 빠르고 안정적으로 된다.

다시 본문으로 돌아와서, 전체 모델은 아래처럼 정리할 수 있다.
이건 그냥 아까 우변의 $f_c$ 라는 함수를 소프트맥스로 해준 것이다. 모델이 내뱉는 확률은 저렇게 계산할 수 있다는 뜻.

여기서 공통적인 특수한 경우는 $f$가 아핀 함수(Affine Function)일 때 나타난다고?

**이건 또 무슨 소리일까?**

앞서 모델이 날것의 로짓을 뱉어내는 함수를 그냥 $f_c$라고 뭉뚱그려 불렀다.

이 함수 안에는 아무 공식이나 넣어도 상관 없지만, 수많은 방법들 중에서 가장 기본적(특수한 경우)으로, 그리고 가장 널리 쓰이는(공통적인) 방식이 1차 방정식 형태인 아핀 함수라는 것이다.

**아핀 함수는 또 뭔데?**

단순한 일차함수라고 생각하자. 이 1차 방정식을 데이터 특성이 여러 개인 다차원 공간으로 확장해서 부르는 수학적 명칭일 뿐이다.

책에서 설명하는 아핀 함수의 형태를 보자.

$$f\left(x\ ;\ \theta \right)\ =\ b+w^Tx=b+w_1x_1+w_2x_2+...+w_Dx_D$$

**여기서 $θ=(b, w)$는 모델의 모수라는데 이 뜻이 뭘까?**

- $θ$ : 앞서 $θ$를 "모델의 뇌 구조 설정값"으로 비유했다. 여기서 이 θ는 "우리 모델이 학습을 통해 찾아내야 할 모든 파라미터를 담아두는 바구니"를 의미한다.
- $(b, w)$ : 우리가 다루고 있는 가장 기본 모델인 이 아핀 함수에서 찾아야 할 핵심 다이얼은 이 2가지, $w$(가중치)와 $b$(편향)이라는 것.
즉, 이 모델에서 찾아야 할 전체 모수 바구니 $θ$ 안에는 $b$와 $w$라는 두 개의 아이템이 들어 있다는 내용을 수학적으로 짧게 표현한 것이다.

그런데 매번 이렇게 긴 수식을 쓸 수는 없으므로 식을 줄여버리고 싶어 하는 듯 하다.

$$f\left(x\right)\ =\ \left(w_1\times x_1\right)+\left(w_2\times x_2\right)+b$$

원래 식은 위와 같다. 여기서 $b$는 $b \times 1$과 같다고 생각하고 식을 풀어보자.

$\tilde{w} = [b, w_1, w_2]$ , $\tilde{x} = [1, x_1, x_2]$ 라고 변형하여 곱하면

$$\tilde{w}^{^T}\tilde{x}=\left(b\times 1\right)+\left(w_1\times x_1\right)+\left(w_2\times x_2\right)$$
이렇게 바꿀 수가 있다.

(여기서 바로 이해가 안가는데, $T$는 transpose, 전치행렬로 곱셈을 위해 모양을 세워주는 것이다. 이렇게 되면 벡터끼리 내적으로 볼 수 있기 때문에 위와 같이 각 요소끼리 더해버릴 수가 있다.)

결과는 같은데 밖으로 튀어나온 $+b$가 행렬 곱셈 안으로 들어간 셈이다.
이걸 책에서는 "가중치 편향 $b$를 가중치 $w$에 흡수시킨다. 따라서 아핀 함수를 선형 함수로 변형한다."는 어려운 표현으로 서술했다.

실제 예를 보자면, 붓꽃 데이터처럼 특성이 4개인 모델의 아핀함수는 다음처럼 생겼을 것이다.
- $x$ : 아까 sl, sw, pl, pw같은 실제 데이터 값이다.

- $w$ (가중치, weight)

각 특성이 정답을 맞히는 데 얼마나 중요한지 나타내는 '중요도'와 같은 값이다.
의미 부여 없이 수식 차원에서 보자면 일차방정식에서의 기울기로 볼 수 있는데, 이러면 왜 미분을 취해 이걸 계산하고 있는지 이해할 수 있다!

- $b$ (편향, bias)

수식으로만 보자면 y절편이 되겠다.

여러가지로 해석해 볼 수 있는데, 데이터가 어디에 분포해 있건 위아래로 움직일 수 있게 하는 장치이기도 하고 아무 입력이 없을 때 기본적으로 내놓는 기본 예측값이라고 볼 수도 있다.

또 신경망 관점에서 본다면 일종의 역치를 정하는 것인데 전반적으로 영점조절을 위한 값이라고 생각하면 편하다.

책에서는 통계학적 편향과 혼동하지 마라고 하고 있다.

여론조사에서 어떤 집단에 편향되어 있다는 식의 그런 의미로 받아들이면 안 된다는 뜻이다.

통계학의 편향은 예측값이 정답에서 한쪽으로 치우쳐져 틀린 정도를 말하므로 완전히 다른 뜻이다.

이 모델을 로지스틱 회귀(Logistic Regression)이라 부른다. (자세한 내용은 나중에 다루는 듯)

그런데 내가 알고 있기로는 소프트맥스는 회귀 모델에서는 취하지 않는다. 근데 로지스틱 회귀라고?

위의 단순한 방정식에서 나온 점수 로짓을 소프트맥스에 통과시켜 확률로 만든 모델을 로지스틱 회귀라고 부르는 것이다.

사실, 이름만 회귀일 뿐 실제로는 분류를 수행하는 알고리즘이니 혼동하지 말자.

참고로 이렇게 다중 분류가 아닌 '개', '고양이'와 같은 이진 분류는 시그모이드(Sigmoid)함수를 씌워 0, 1로 구별한다. 알아만 두자.

그럼 이걸 왜 '회귀'라고 할까? 중간 결과물의 형태에 집중해서 명명했기 때문이다.

선형 회귀처럼 연속적인 숫자를 예측하는 것을 회귀라 부르는데, 이 모델이 수식을 계산해서 뱉어내는 값은 분류된 클래스가 아니라

"$n$번 클래스일 확률은 $p$입니다"라는 연속적인 숫자로 말하고 있기 때문이다.

특정 클래스에 속할 확률(연속적인 값)을 회귀식으로 예측하기 때문에 수학적인 족보 상 '회귀'라는 이름이 붙은 것이다.

참고로 로지스틱 회귀는 타겟 변수 $y$가 0 또는 1의 값을 가지는 베르누이 분포를 따른다고 가정한다.

이 가정 하에 MLE를 전개하고 로그를 씌우면 BCE(Binary Cross-Entropy) 손실 함수가 도출되고, 이게 NLL 최소화 과정과 동일한 과정이다.

## 최대 가능도 추정 / Maximum Likelihood Estimate

1. **좋은 모델이란 무엇인가?**

확률적 모델에서 $f(x ; θ)$는 파라미터 $θ$를 가진 모델에 입력 $x$를 넣었을 때의 결과이다.

그리고 $p( y | f( x ; θ ) )$는 "모델이 예측하기에 실제 정답이 $y$일 확률"을 의미한다.

예로, 정답인 고양이 사진($x$)를 보여줬을 때, 이 사진이 고양이($y$)일 확률을 99%(0.99)로 예측했다면 아주 좋은 모델이다. 참인 출력 $y$에 높은 확률을 할당할수록 좋은 모델이다.

2. **왜 음수(-)와 로그를 사용할까?**\

$$\ell \left(y\ ,\ f\left(x\ ;\ \theta \right)\right)=-\log p\left(y\mid f\left(x\ ;\ \theta \right)\right)$$

이 수식은 손실 함수(Loss Function)을 나타낸다.

손실(오차)는 작을수록 좋다. 하지만 우리가 구한 확률은 클 수록 좋다.
이 반대되는 성질을 맞춰주기 위해 사용하는 것이 바로 -log이다.

그렇다면 왜 굳이 log일까?

확률은 0과 1사이의 값이다. 데이터가 많아질수록 이 확률들을 전부 곱해야 하는데 0.xxx로 가는 확률들을 계속 곱하다 보면 숫자가 너무 작아져서 0으로 수렴하는 꼴(언더플로우)이 된다.

로그를 씌우면 곱셈이 덧셈으로 바뀌어 계산이 훨씬 쉬워진다. (밑이 같은 로그끼리는 더할 수 있으니까)

또, $-$로 하는 이유는 뭘까?
정답 확률이 100%(1.0)이라고 해 보자. 이 경우 $-log(1.0) = 0$ 이 된다.
즉, 완벽하게 맞추면 손실이 0이 되는 구조를 만들어 준다.

확률이 낮아져 0에 가까워질수록 $-log$ 값은 무한대로 커지며, 엄청난 손실을 주게 된다.

위 그래프를 보면 0에 가까워질수록 발산하고 1에서 0이 됨을 볼 수 있다.

3. **음의 로그 가능도, NLL (Negative Log Likelihood)**

$$\text{NLL}\left(\theta \right)=-\frac{1}{N}\sum _{n=1}^N\log p\left(y_n\mid f\left(x_n\ ;\ \theta \right)\right)$$

이 수식은 모델이 가진 전체 데이터(1~N)에 대해 위에서 구한 손실(-log)들의 평균을 낸 것이다.
훈련 세트 전체를 기준으로 모델이 얼마나 틀리고 있는지를 나타내는 지표로, 이를 음의 로그 가능도(NLL)라고 부른다.

4. **최대 가능도 추정값, MLE (Maximum Likelihood Estimate)**

$$\hat{\theta }_{\text{mle}}=\arg \min _{\theta }^{ }\ \text{NLL}\left(\theta \right)$$

arg min이 뒤에 나오는 함수 값을 최소화하는 θ를 찾으라는 것이다.
즉, 훈련 데이터 전체에 대한 손실(NLL)을 가장 최소화하는 최적의 θ를 찾는 과정을 보여준다.
이 과정이 모델을 데이터에 적합시키는 매우 일반적인 방법이라고 소개하고 있다.

## 회귀 / Regression

1. **회귀와 이차손실 등장**

지금까지는 붓꽃이 setosa인가 virginica인가를 맞혔다면(분류), 이제는 "이 꽃의 독성 수치가 7.5인가? 식물의 높이가 15.2cm인가?" 처럼 연속적인 실숫값($y∈ℝ$)을 맞히는 문제로 넘어왔다. 이를 **회귀**라고 부른다.

출력값이 연속된 숫자이다 보니, 앞서 분류에서 썼던 "0점 아니면 1점"의 채점 방식을 쓸 수 없다.

그래서 등장한 채점 기준이 **이차 손실(L2 Loss)**이다.

$$\ell _2\left(y,\ \hat{y}\right)=\left(y-\hat{y}\right)^2$$

여기서 $y - \hat{y}$ (잔차, Residual)는 정답($y$)에서 모델의 예측값($\hat{y}$)을 뺀 단순한 '오차 거리'이다.

**왜 굳이 제곱을 할까?**

오차가 양수든 음수든 무조건 +로 벌점을 주기 위해서, 큰 오차에 더 큰 불이익을 주기 위해서이다.

오차가 1이면 벌점 1점 이지만, 오차가 4로 벌어지면 벌점은 16으로 폭증한다.

즉, 모델에게 "실수는 봐 주지만 헛소리는 절대 하지 마!"라고 하는 꼴이다.

2. **경험적 위험(Empirical Risk)와 평균 제곱 오차(Mean Square Error / MSE)**

앞서 나왔던 경험적 위험은 전체 학습 데이터에 대해 모델이 평균적으로 얼마나 틀리고 있는지를 나타내는 지표라고 했다. 즉, 전체 데이터에서 발생한 Loss들의 평균이다.

$$\mathrm{MSE}=\frac{1}{n}\sum _{i=1}^n\left(y_i-\hat{y_i}\right)^2$$

수식으로 보면 이러한데, 저 기호들을 지금까지 나왔던 기호들로 치환해서 생각하면 이해가 쉽다.

앞서 잔차(residual)이 오차 거리라고 하였고 식에서의 시그마 안의 항에 해당한다. 오차들의 평균을 낸 그것이 MSE라는 소리인데, 이는 경험적 위험과 같은 방식이다.

정확히 말하면 경험적 위험은 일반적인 개념이다. 이 일반적인 방식에서의 오차에 해당하는 부분에 L2 Loss를 대입한 것이 MSE이다.

(계속 햇갈리는데, 경험적 위험은 수식에서의 $\frac{1}{N} Σ$에 해당하는 개념으로, 개별 데이터들이 정답에 비해 얼마나 떨어져 있냐를 평균내는 방식 자체를 뜻한다고 생각하자.)

3. **불확실성과 정규분포(Gaussian Distribution)**

회귀에서도 모델은 완벽한 숫자를 찍을 수 없다.

그래서 "보니까 높이는 15cm 정도인데, 거기서 약간의 오차가 있을 확률이 높아"처럼 대답해야 한다.

이 때 오차가 분포하는 모양을 자연계에서 가장 흔한 정규분포를 따른다고 가정한다. (평균에 모인 종 모양, 가우시안 분포라고 하면 정규분포를 뜻한다)

$$\mathcal{N}(y \mid \mu, \sigma^2) \triangleq \frac{1}{\sqrt{2\pi\sigma^2}} e^{-\frac{1}{2\sigma^2}(y-\mu)^2}$$

위 식이 정규분포인데... 하나 하나 살펴보자.

1) $μ$ (평균) : 종 모양 그래프의 중심. 회귀에서는 모델의 예측값 f(x ; θ)가 바로 이 중심이 된다.

2) $σ^2$ (분산) : 종의 퍼진 정도(불확실성)이다.

3) $1\sqrt{}$ : 몰라도 된다. 이건 그냥 그래프 전체 넓이를 1(100%)로 맞춰주기 위한 비율 조정용 상수일 뿐이다.

3. **NLL와 MSE의 비례성**

모델 학습에서 음의 로그 가능도(NLL)를 최소화하는 방식을 사용한다고 했다.

여기서 한 번 생각해보자. 저 위의 정규분포 수식에 $log$를 씌우면 어떤 일이 일어날까?

밑이 $e$인 로그를 씌운다고 했을 때 $e$는 사라지고 지수만 뚝 떨어진다.

저기 있는 $μ$ 도 $f(x ; θ)$와 같은 의미라고 하였으므로 결론적으로는 다음과 같은 항이 떨어진다.

$$\frac{1}{2\sigma ^2}\left(y-f\left(x\ ;\ \theta \right)\right)^2$$

다시 한번 더 돌아가서 $f(x ; θ)$은 예측값으로, L2 Loss에서 본 $\hat{y}$ 동일한 항이다.
그러므로 저 분수에 곱해진 제곱 항은 L2 Loss 자체가 된다. 이들을 정리해 복잡한 상수들을 치워버리고 식을 정리하면 이렇게 된다.

$$\mathrm{NLL}\left(\theta \right)=\left(\frac{1}{2\sigma ^{^2}}\right)\mathrm{MSE}\left(\theta \right)+상수$$

예측 오차가 정규분포를 따른다고 가정하고, 가장 확률이 높은 정답(최대 가능도 계산=NLL최소화)을 찾아보았더니 오차와의 거리(MSE)를 가장 작게 만드는 방식과 동일하다는 것을 볼 수 있다.
(MSE가 가장 높은 확률의 정답을 찾는 방식에 적합하다는 것을 증명한 것이다)

이렇게 오차를 가우시안 분포를 따른다고 가정하는 것이 선형 회귀(Linear Regression)이다.

## 선형 회귀

$θ$를 조정하여 최소 제곱 해(least squares solution)를 찾을 때까지 오차 제곱 합을 최소화할 수 있다.
입력 특성이 여러 개라면 다음과 같이 쓸 수 있다.
여기서 $θ = (w, b)$이다. 이걸 다중 선형 회귀라고 부른다.
